업무 상황별 프롬프트 템플릿 목록
═══════════════════════════════════════════════
※ 상황에 맞는 템플릿을 골라 [괄호] 안만 바꿔 쓰세요!

───────────────────────────────────────────────
📂 상황 1: 파일/폴더 정리
───────────────────────────────────────────────
· 정리 기본형:
  [폴더경로]의 파일을 [분류기준(예: PDF/엑셀/이미지)] 폴더로 분류해줘. 
  이동 전 계획을 표로 보여주고, 원본은 복사로.

· 이름 변경형:
  [폴더경로]의 [파일종류]를 [이름규칙(예: 행사_001)]부터 순서대로 이름 바꿔줘. 
  백업 폴더를 먼저 만들어 원본 복사 후 진행. 변경 계획 표로 먼저.

· 검색형:
  [폴더경로]에서 [조건(예: 이름에 '보고서' 포함 / 30일 이상 지난)]인 파일을 찾아 목록으로 보여줘.

· 백업형:
  [폴더경로]의 [파일종류]를 '백업_[오늘날짜]' 폴더에 복사하고 zip으로 압축해줘.

───────────────────────────────────────────────
📊 상황 2: 엑셀/CSV 집계
───────────────────────────────────────────────
· 요약형:
  [파일경로]를 열어서 어떤 항목이 있고 총 몇 건인지 요약해줘.

· 집계형:
  [파일경로]에서 [항목]별 [합계/인원수/평균]을 구해줘. 
  [정렬기준] 순으로 정렬해서 새 파일 '[파일명].xlsx'로 저장. 원본은 그대로.

· 청소형:
  [파일경로]에서 빈칸·중복 행을 제거하고 '[정리완료파일명].csv'로 저장해줘. 
  원본은 건드리지 말고, 정리 전후 행 수를 알려줘.

· 병합형:
  [파일1]과 [파일2]를 [기준열] 기준으로 합쳐 하나의 표로 만들어줘. 
  새 파일로 저장.

───────────────────────────────────────────────
📈 상황 3: 분석/차트
───────────────────────────────────────────────
· 교차표형:
  [파일경로]에서 행은 [항목A], 열은 [항목B]으로 [값] 교차표를 만들어줘.

· 차트형:
  [데이터]를 [차트종류(막대/꺾은선/원)] 차트로 만들어 엑셀 파일에 넣어줘. 
  차트 제목은 '[제목]', 파란 톤으로.

· 보고 요약형:
  [분석 결과]를 상사 보고용 3줄 요약으로 정리해줘. 
  핵심 숫자·잘 된 점·제안 포함.

───────────────────────────────────────────────
📝 상황 4: 문서/보고서 생성
───────────────────────────────────────────────
· 메일머지형:
  [명단파일]로 [문서종류(안내문/수료증/고지서)]을 한 명당 1장씩 만들어줘. 
  [삽입항목(이름·등급·날짜)] 자동 삽입. 전체를 한 PDF로도 합쳐줘.

· 보고서형:
  [데이터파일]로 '[보고서 제목]'을 만들어줘. 
  표지(제목·작성일·작성자) + 본문 표 + 3줄 요약 포함, PDF 저장.

· 다듬기형:
  이 보고서의 [특정 부분]을 [어떻게] 수정해줘.
  (예: "표 아래에 코멘트 추가", "글자 수를 줄여")

───────────────────────────────────────────────
🎤 상황 5: 발표 자료
───────────────────────────────────────────────
· PPT 생성형:
  [보고서/데이터]를 참고해 발표 슬라이드를 만들어줘. 
  표지·목차·본문([N]장)·요약·마무리 구조. 하단에 회사명·쪽번호.

· PPT 다듬기형:
  슬라이드 [배경색/글자크기/색감]을 [요구사항]으로 통일해줘.
  본문은 장당 [N]줄 이내로 줄여줘.

· 대본형:
  각 슬라이드 발표 멘트를 슬라이드당 [N]초 분량으로 만들어줘. 
  [말투(존대/친근)] 말투, 다음 장 연결 멘트 포함.

· 패키지형:
  PPT + 대본 + 체크리스트를 '[폴더명]'에 모아 zip으로 압축해줘.

───────────────────────────────────────────────
📄 상황 6: PDF 처리
───────────────────────────────────────────────
· 합치기형:
  [폴더/파일들]을 [순서]대로 '[결과파일명].pdf'로 합쳐줘.

· 나누기형:
  [PDF파일]을 [기준(페이지별/특정페이지)]으로 나눠줘.

· 추출형:
  [PDF파일] 내용을 [텍스트/표→엑셀]로 뽑아줘.
  '[키워드]'가 들어간 문장은 별도로 정리해줘.

· 워터마크형:
  [PDF파일]에 '[텍스트(대외비/샘플)]' 워터마크를 넣어줘.

───────────────────────────────────────────────
🔍 상황 7: 검토/확인
───────────────────────────────────────────────
· 비교형:
  [파일A]와 [파일B]의 [항목] 차이를 비교해서 표로 보여줘.

· 검증형:
  [파일]에서 [조건(빈칸/중복/오류)]이 있는지 확인하고 결과를 알려줘.

· 요약형:
  [파일/작업결과]를 [형식(표/3줄/1페이지)]으로 요약해줘.

───────────────────────────────────────────────
🔄 상황 8: 단계 나누기
───────────────────────────────────────────────
· 단계 분리형:
  [작업]을 할 거야. 1단계([작업1])만 하고 결과 보여줘.
  → (확인 후) 2단계([작업2])만 이어서 해줘.
  → (확인 후) 3단계([작업3])만 해줘. 앞 단계 결과 그대로 활용.

· 조건부형:
  [조건]이면 [작업A], 아니면 [작업B]로 해줘.

───────────────────────────────────────────────
